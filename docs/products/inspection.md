Bus Inspection is the BusDK product line for local inspection reports and
action lists. It gives inspection teams workspace-local workflows for sites,
observations, photos, customer acknowledgements, role-scoped access, generated
reports, and action-list follow-up.

Use this product when inspection data, report snapshots, photos, customer
responses, and recurring follow-up need to stay connected to one local
workspace.

## Start Here

1. Read [`bus-inspection`](../modules/bus-inspection) for the module-level
   command and product surface.
2. Read the UI integration docs under [product module integration](../ui/fc-025-product-module-integration/)
   when evaluating portal-hosted module surfaces.
3. Use [deployment and data control](../integration/deployment-and-data-control)
   when deciding where inspection data and exports should live.

## Product Modules

The product is centered on [`bus-inspection`](../modules/bus-inspection). It
uses supporting platform capabilities from Bus Gateway, Bus UI, workspace data,
document/export infrastructure, and Bus AI Platform when AI-assisted
configuration needs a model adapter.
