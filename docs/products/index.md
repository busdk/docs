BusDK is documented as a product family. Start with the product line that
matches the work you are trying to do, then use the module reference for exact
commands, flags, and provider ownership.

## Main Product Lines

- [Bus Agentic Development](agents): semi-autonomous software development with
  task threads, workers, agent runtime, repositories, MCP, notes, terminal
  state, review, and local or SSH-accessible work capacity.
- [Bus AI Platform](ai-platform): self-hostable AI product infrastructure with
  model access, inference runtime control, deployment, auth, usage, billing,
  VM/container/terminal runtime support, and service hooks.
- [Bus Books](books): auditable financial workflows over inspectable workspace
  data for humans, CLI/API automation, and agentic AI.
- [Bus Inspection](inspection): local inspection reports, action lists, photo
  evidence, customer acknowledgement, access control, and snapshot exports.
- [Bus GX/UI Library](gx-ui): Go-native UI components with TSX-like authoring,
  compiled Go render roots, reusable components, runtime bridges, and tests.
- [Bus Top](top): local process monitoring with deterministic system facts,
  process-family grouping, snapshots, diagnosis, and optional AI explanations.
- [Bus Services](services): predictable project service stacks from
  `services.yml` without requiring virtualization.
