# bus-notes

`bus notes` is the Bus command for structured project notes. It is intended for
engineering diary entries, agent session memos, decisions, task handoffs,
release notes, investigations, and lessons learned.

Bus Notes are not raw debug logs. Notes should be safe to share according to
their visibility and should never include secrets, tokens, credentials, private
customer data, or unredacted runtime environments.

Planned commands include `init`, `add`, `list`, `show`, `search`, `validate`,
`import`, and `export`.
